【日常】My First Blog

· 2023-03-23 · # 日常

Re:0 从零开始

自从2018年我接触到服务器之后，就一直想要搭建一个完全属于自己的博客，最初的想法是作为自己的一个树洞，把不想在公众面前发布的内容 （或者是一些法律禁止的东西） 发表在这里。于是我也尝试了不少解决方案：Wordpress、Typecho、Zblog等传统博客系统；以及Hugo、Jekyll等静态博客，包括本次选择的Hexo。在折腾的时候发现这些系统都多少存在一些问题，也导致我在各种系统间反复跳跃。

  • 传统博客
    传统博客存在的问题众所周知，即“臃肿”、“功能复杂”、“依赖服务器环境运行”等……不过在我拥有甲骨文永久免费服务器之后，这些问题就没有那么严重了。于是我用了比较长时间的Typecho，并写了不少文章。但是永久免费终究是不知道能用多久，因此我一直在寻找其他的托管平台。

  • 静态博客
    在使用静态博客的时候，对我来说最大的问题就是环境配置。上网找了许多教程，基本都是说“先这样、再这样，然后就好了”，到我实操就变成 “这样就报错，那样还是报错,到底该怎么样”。所以我上面说使用Typecho其实就是在【易用】与【可持续】之间做了个取舍。

  • xLog & Gridea
    这两者是主打“托管 + 写作”一体的平台，xLog是基于区块链 (韭菜园) 存储，Gridea是传统服务器托管 + 付费的模式。
    xLog我用了一下，发现一个对我来说比较大的痛点，那就是它的markdown编辑器容易崩溃，且使用时需要登录区块链钱包……我只是想写写东西，并不想扯上什么区块链、虚拟货币之类的东西。同时xLog的分类功能堪称鸡肋，页面整体属于信息流模式，也不能搜索，对我来说使用起来比较麻烦。
    Gridea就算了，付费写博客我是真干不出来，毕竟也不做站，免费的话又是需要服务器托管……

Hello World

兜兜转转，这些年也看了不少人的博客，发现 Hexo + Github + Cloudflare Pages 的配置真的非常好用省心，于是最终还是回来尝试一下。如果各位能看到这篇文章，那么就代表我成功配置了这套系统。借助Cloudflare的全球基础建设，相信本博客也能活的更久。

改变

Hexo 固然有数据在本地的安心感，但对我这种经常折腾系统的人来说，nodejs + hexo + vscode + 设置proxy 的配置意味着提高了每次装机的设置难度，再加上我手里的服务器又这么多，因此最终还是决定回归初心，将博客放在甲骨文的服务器上了。

更新

  • 2025/05/28： 由于Github对于IP封控趋于严格，导致博客中许多图片报错429，故正在将博客中所有图片替换为微博图床链接。
  • 2026/01/09： 开始尝试使用Gridea写作，直接同步到Github Pages，避免了各种同步问题
